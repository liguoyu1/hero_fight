# Phase 04: 完善打磨 — PLAN.md

## 目标

代码规范化 + 性能微调 + 视觉效果增强，共 10 项需求。

## Wave 1: 代码规范 + 错误处理

### Task 4.1: 注释语言统一 + 异常处理

**问题:** 中文/英文注释混用；`catch(_){}` 多处吞异常。

**方案:**
1. 所有注释统一为英文
2. 所有 catch 块添加 `debugPrint('Error in <context>: $e')`
3. 关键路径（网络连接、数据加载）添加 try-catch + 用户可见错误提示
4. `fromJson` 方法添加类型安全检查（`is num` 替代强制 `as num`）

**文件:** `lib/game/fighter_game.dart`, `lib/game/components/fighter.dart`, `lib/network/game_client.dart`, 等

### Task 4.2: 渲染裁剪

**问题:** 离屏 fighter/projectile/particle 仍被完整渲染。

**方案:**
1. 在 `Fighter.render()` 开头添加：`if (!camera.visibleRect.overlaps(boundingBox)) return;`
2. 在 `ParticleSystem.render()` 中仅渲染屏幕范围内的粒子
3. 在 `Projectile.render()` 中添加可见性检查

**文件:** `lib/game/components/fighter.dart`, `lib/game/components/particle_system.dart`, `lib/game/components/projectile.dart`

## Wave 2: 视觉效果

### Task 4.3: 地面阴影

**问题:** 英雄脚下无阴影，悬浮感强。

**方案:**
1. 在 `HeroRenderer` 或 `Fighter.render()` 中添加椭圆阴影
2. 阴影大小随跳跃高度变化（越高越小越淡）
3. 使用 `canvas.drawOval()` + 半透明灰色

**文件:** `lib/game/components/hero_renderer.dart`, `lib/game/components/fighter.dart`

### Task 4.4: 单眼渲染修复

**问题:** 左右翻转变换导致只画一只眼。

**方案:**
1. 眼球绘制代码中添加 `canvas.save()` / `canvas.restore()` 隔离翻转影响
2. 或在眼球绘制前检查 `flipX` 状态单独处理

**文件:** `lib/game/components/hero_renderer.dart`

## Wave 3: 移动端体验

### Task 4.5: 触觉反馈

**问题:** 移动端无 haptic feedback。

**方案:**
1. 攻击命中：`HapticFeedback.mediumImpact()`
2. 技能释放：`HapticFeedback.heavyImpact()`
3. 受击：`HapticFeedback.lightImpact()`
4. 使用 `HapticFeedback` 类（Flutter services 内置）

**文件:** `lib/game/fighter_game.dart`, `lib/game/components/fighter.dart`

### Task 4.6: 触摸控件优化

**问题:** 摇杆透明度极低（base 0x44 / knob 0x88），可能难以辨识。

**方案:**
1. 摇杆不透明度从 0x44/0x88 提升至 0x66/0xBB
2. 按钮添加按压时缩放反馈（`scale: 0.9` 过渡）

**文件:** `lib/game/components/touch_controls.dart`

## Wave 4: UI 完善

### Task 4.7: 排行榜 UX

**问题:** 无 loading 状态；第 4 名起显示数字而非序数。

**方案:**
1. 添加 `_isLoading` 状态 + `CircularProgressIndicator`
2. 名次格式化：1→🥇, 2→🥈, 3→🥉, 4+→"4th"/"5th"...
3. 错误状态显示重试按钮

**文件:** `lib/screens/stats_screen.dart`

### Task 4.8: 教程可重播

**问题:** `resetTutorial()` 已实现但未暴露到 UI。

**方案:**
1. 主菜单设置中添加「Replay Tutorial」按钮
2. 点击调用 `resetTutorial()` + 导航到教程页

**文件:** `lib/screens/main_menu.dart`, `lib/game/components/tutorial_overlay.dart`

### Task 4.9: 弹体墙壁自适应

**问题:** `projectile.dart` 墙壁常量硬编码 1280×600。

**方案:**
1. 弹体接收 `stageWidth` 和 `groundY` 参数（从 FighterGame 传入）
2. 移除硬编码的 `wallLeft/wallRight/groundY` 常量
3. 弹体边界改为「撞击消失」而非「环绕重入」

**文件:** `lib/game/components/projectile.dart`, `lib/game/fighter_game.dart`

## Wave 5: 音频 + 网络微调

### Task 4.10: 音效机制修复

**问题:** 跳跃音效每帧触发；playProjectile/playLand 定义但未调用。

**方案:**
1. 跳跃音效：改为 `onJumpPressed()` 单次触发（在状态转换时）
2. 弹体音效：`Projectile.spawn()` 时调用 `SoundManager().playProjectile()`
3. 落地音效：跳跃落地检测时调用 `SoundManager().playLand()`

**文件:** `lib/game/components/fighter.dart`, `lib/game/components/projectile.dart`, `lib/game/audio/sound_manager.dart`
