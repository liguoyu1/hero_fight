# Phase 03: 体验提升 — PLAN.md

## 目标

国际化全覆盖 + AI 强化 + 网络自适应 + 命名统一 + 游戏平衡微调

## Wave 1: 国际化全覆盖 + 游戏平衡

### Task 3.1: i18n 全覆盖

**问题:** 英雄选择页（Ready/PICKED/阵营名）、主菜单设置、教程覆盖层硬编码英文。

**方案:**
1. `hero_select.dart`: Ready/PICKED/P1/P2/AI/You/Opponent/阵营名 → `AppLocalizations`
2. `main_menu.dart`: 设置对话框标题/按钮 → i18n
3. `tutorial_overlay.dart`: 4 步骤标题和描述 → i18n（键已定义，切换 const 为动态读取）
4. `stats_screen.dart`: "Play N more games" / "Need at least 5 games" → i18n
5. 补充 `app_localizations.dart` 和 `app_localizations_en.dart` 中缺失的键

**文件:** `lib/screens/hero_select.dart`, `lib/screens/main_menu.dart`, `lib/game/components/tutorial_overlay.dart`, `lib/screens/stats_screen.dart`, `lib/i18n/app_localizations.dart`

### Task 3.2: 后羿技能平衡

**问题:** 后羿 2 秒冻结 = 命中即必杀。

**方案:**
1. 冻结时间从 2.0s 改为 0.8s + 递减（第二次命中冻结 0.5s，第三次 0.3s）
2. 添加 `freezeCount` 计数器，5s 后重置
3. 弹体速度降低 20%

**文件:** `lib/game/heroes/ashe.dart`

### Task 3.3: 连击输入缓冲

**问题:** 攻击按钮仅帧精确按下时生效，触屏难以完成连击。

**方案:**
1. 在 `Fighter._processInput()` 中添加 `_inputBuffer` 队列（3-5 帧缓冲）
2. 按下攻击键后，在缓冲窗口内检测到 → 执行连击下一段
3. 缓冲过期清除

**文件:** `lib/game/components/fighter.dart`

## Wave 2: AI 强化 + 网络自适应

### Task 3.4: AI 方向攻击 + 弹体躲避

**问题:** AI 只会设置 `fighter.input.attack = true`，从不使用方向攻击；不对弹体做出反应。

**方案:**
1. 方向攻击决策：根据距离/位置选择 upAttack/fwdAttack
2. 弹体躲避：检测接近的弹体 → 向远离方向移动或跳跃
3. 修复「两个优先级 6」的 bug

**文件:** `lib/game/ai/ai_controller.dart`

### Task 3.5: 网络 RTT 自适应

**问题:** `delayFrames` 固定 2 帧，无 RTT 测量。

**方案:**
1. 客户端发送 ping 帧，服务端回复 pong → 计算 RTT
2. `delayFrames = max(2, ceil(RTT_ms / 16))` 自适应
3. 每 5s 重新测量一次
4. 网络质量差时提升 delayFrames 以降低回滚频率

**文件:** `lib/game/network/rollback_engine.dart`, `lib/network/game_client.dart`, `server/index.js`

## Wave 3: 代码规范统一

### Task 3.6: 英雄命名统一

**问题:** 拼音(Lubu) vs 英文(ShieldGeneral) vs LOL遗留(lee_sin→少林武僧)。

**方案:**
1. 文件名改为语义化英文：`lubu.dart` → `lu_bu.dart`（保持辨识度）、`lee_sin.dart` → `shaolin_monk.dart`
2. 类名统一风格：`LubuHero` → `LuBuHero`（保持）、`ShieldGeneralHero` → `ShieldGeneralHero`（不变）
3. 更新所有 import 引用
4. `HeroRegistry` 中 heroId 与文件名一致

**文件:** `lib/game/heroes/*.dart`, `lib/game/heroes/hero_registry.dart`, `lib/game/heroes/hero_loader.dart`, `lib/game/heroes/all_heroes.dart`

### Task 3.7: GameClient 事件系统重构

**问题:** 17 个独立 StreamController，大量样板代码。

**方案:**
1. 定义 `GameEvent` 枚举（connected/disconnected/onRoomCreated/...等 17 种事件）
2. 用单个 `StreamController<GameEvent>` 替代 17 个
3. 提供 `on<T>(GameEvent event)` 类型安全的监听方法

**文件:** `lib/network/game_client.dart`

### Task 3.8: HeroLoader OCP 修复

**问题:** 硬编码 switch-case 映射 heroId → HeroData 构造函数，新增英雄需改 3 处。

**方案:**
1. 利用 `HeroRegistry` 的注册表：添加 `HeroRegistry.create(String heroId)` 方法
2. `HeroLoader.loadHero(String heroId)` 调用 `HeroRegistry.create(heroId)`
3. 删除硬编码 switch-case
4. 新增英雄仅需在 hero 文件中注册 + HeroRegistry 添加（2 处）

**文件:** `lib/game/heroes/hero_loader.dart`, `lib/game/heroes/hero_registry.dart`
