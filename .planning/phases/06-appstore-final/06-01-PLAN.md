# Phase 06: App Store 最终审核 — PLAN.md

## 目标

满足 App Store 审核人员模拟中发现的 6 项缺失要求。

## Wave 1: 政策文件 + 安全加固

### Task 6.1: 创建隐私政策页面

**问题:** App Store 要求可访问的隐私政策 URL。

**方案:**
1. 创建 `web/privacy.html` — 中文+英文双语隐私政策
2. 声明数据采集：设备 ID（用于战绩匹配）、昵称（用户自定义）
3. 声明不追踪、不分享给第三方
4. 提供联系方式
5. 部署到 GitHub Pages 或托管在应用内 web/ 目录

**文件:** `web/privacy.html`

### Task 6.2: 创建支持页面

**问题:** App Store 要求技术支持 URL。

**方案:**
1. 创建 `web/support.html` — 包含 FAQ + 联系方式
2. 或直接使用 GitHub Issues 链接

**文件:** `web/support.html`

### Task 6.3: appSecret 安全加固

**问题:** `hero-fighter-secret-key-2024` 硬编码在二进制中。

**方案:**
1. `app_config.dart`: 移除默认值字符串，改为 `null`
2. 添加注释：生产环境通过环境变量/安全存储注入
3. 程序启动时若检测到 null 且非 debug 模式，打印警告

**文件:** `lib/config/app_config.dart`

### Task 6.4: 服务端部署确认 + 网络断开友好提示

**问题:** 审核时若服务端不可用，App 应友好提示。

**方案:**
1. `game_client.dart`: connect() 超时后添加友好错误而非仅 `debugPrint`
2. 确保 `production` 环境指向已部署的 Railway URL

**文件:** `lib/network/game_client.dart`

### Task 6.5: App 图标质量提升

**问题:** 非 Retina 图标为 8-bit colormap PNG。

**方案:**
1. 使用 Python/Pillow 重新生成真彩色 PNG 图标
2. 确保所有尺寸为 32-bit RGBA

**文件:** `ios/Runner/Assets.xcassets/AppIcon.appiconset/*.png`

### Task 6.6: IPv6 兼容

**问题:** App Store 审核网络为 IPv6-only。LAN 发现用 `udp4` 可能失败。

**方案:**
1. UDP socket 创建时尝试 `udp6` fallback
2. 或添加 try-catch 确保 IPv6 网络下不崩溃（静默降级）

**文件:** `lib/network/lan_discovery_io.dart`
