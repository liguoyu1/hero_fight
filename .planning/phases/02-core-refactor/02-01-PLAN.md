# Phase 02: 核心重构 — PLAN.md

## 目标

解决架构问题 + 游戏平衡 + 安全加固，共 11 项需求。

## Wave 1: 架构重构

### Task 2.1: FighterGame 拆分

**问题:** 1167 行单文件混合输入处理/相机控制/HUD/VFX/轮询管理。

**方案:**
1. 提取 `InputSystem` — 处理 keyboard/touch 输入到 FighterInput 映射
2. 提取 `CameraController` — 相机跟随/边界/缩放逻辑
3. 提取 `VFXSystem` — 粒子生成/弹体管理/特效协调
4. `FighterGame` 保留核心游戏循环 + 组件注册

**文件:** 新建 `lib/game/systems/input_system.dart`, `camera_controller.dart`, `vfx_system.dart`

### Task 2.2: Fighter 拆分

**问题:** 875 行混合渲染（300 行 switch-case 技能绘制）+ 状态机 + 碰撞 + 序列化。

**方案:**
1. 提取 `FighterRenderer` — 将所有 Canvas 绘制代码（含技能视觉效果）移入
2. `Fighter` 保留状态机、碰撞检测、业务逻辑
3. 渲染时 `FighterRenderer.render(canvas, fighter)` 被 FighterGame 调用

**文件:** 新建 `lib/game/renderers/fighter_renderer.dart`

### Task 2.3: 服务端模块化

**问题:** 525 行 index.js 混合 WebSocket 处理/房间管理/匹配队列/心跳/UDP 发现。

**方案:**
1. `ws-handler.js` — WebSocket 消息分发 + 客户端生命周期
2. `room-manager.js` — 房间 CRUD/状态机/房主转移
3. `matchmaker.js` — 匹配队列/MMR/匹配确认
4. `index.js` 仅保留 HTTP 服务器创建 + 模块组装

**文件:** 新建 `server/ws-handler.js`, `server/room-manager.js`, `server/matchmaker.js`

## Wave 2: 游戏平衡 + 系统修复

### Task 2.4: 英雄属性同步

**问题:** `executeSkill()` 硬编码伤害与 `skillDamage` 属性不一致。

| 英雄 | skillDamage | executeSkill 硬编码 | 偏差 |
|------|-------------|---------------------|------|
| 吕布 | 240 | 250 | +10 |
| 关羽 | 280 | 300 | +20 |
| 蚩尤 | 320 | 350 | +30 |
| 墨家 | 260 | 280 | +20 |

**方案:** 所有英雄的 `executeSkill()` 读取 `skillDamage` 属性而非硬编码。

**文件:** `lubu.dart`, `guanyu.dart`, `thanos.dart`, `mohist.dart`

### Task 2.5: 跳跃系统实现

**问题:** `input.jump` 仅设置 `moveY -= 1.5`（无重力/跳跃力），`jumpForce` 属性未使用。

**方案:**
1. 新增 `velocityY` 字段（初始 0）
2. 跳跃：`velocityY = -jumpForce`（一次性脉冲）
3. 每帧 `velocityY += gravity * dt`（重力加速度）
4. 落地检测 `position.y >= groundY` → 重置 velocityY
5. 移除 `input.jump` 持续向上的行为

**文件:** `lib/game/components/fighter.dart`, `lib/game/fighter_game.dart`

### Task 2.6: 安全加固

**问题:** 硬编码密钥 `hero-fighter-secret-key-2024`，无 game_input 合法性校验。

**方案:**
1. 客户端: `AppConfig.appSecret` 从环境变量/配置读取
2. 服务端: `APP_SECRET` 从 `process.env.APP_SECRET` 读取，fallback 仅用于开发环境并打印警告
3. 服务端 game_input 处理：验证 frame 递增、输入字段为 bool
4. 添加 `.env.example` 文件

**文件:** `lib/config/app_config.dart`, `server/index.js`, `server/.env.example`

### Task 2.7: ParticleSystem 性能优化

**问题:** O(n) 线性扫描获取粒子（480 次循环/技能爆发）；activeCount 每帧重算。

**方案:**
1. 维护 free-list（链表/栈）实现 O(1) acquire
2. activeCount 增量维护（acquire +1, deactivate -1）
3. 添加 maxPoolSize=128 上限 + 超限日志警告
4. `pow(friction, dt)` 改为预计算常数

**文件:** `lib/game/components/particle_system.dart`

### Task 2.8: canvas.saveLayer 替代

**问题:** 死亡淡出用 `canvas.saveLayer` 每帧分配离屏缓冲区。

**方案:** 在绘制死亡角色前设置 `paint.color = Color.fromRGBO(255, 255, 255, deathAlpha)` 而非 saveLayer 整体合成。

**文件:** `lib/game/components/fighter.dart`

### Task 2.9: 死代码清理

**问题:** HeroLoader 加载 JSON 但从未解析使用；7 个 WAV 文件 (~130KB) 在 bundle 中但 audio 系统用程序化合成。

**方案:**
1. 移除 `HeroLoader.loadHeroJson()` 和 `_jsonCache`
2. 保留 WAV 文件（Task 1.3 将启用真实播放）
3. 检查并移除其他未使用代码

**文件:** `lib/game/heroes/hero_loader.dart`

## Wave 3: 网络增强

### Task 2.10: 重连状态恢复

**问题:** 重连仅恢复 WebSocket，游戏状态丢失。

**方案:**
1. 客户端以 deviceId（非 ws 对象）为 key 注册
2. 服务端保留断线玩家状态（30s 宽限期）
3. 重连后发送 `request_resync` → 服务端回复当前房间/游戏状态
4. 客户端根据状态恢复 UI（重新加入房间/观战）

**文件:** `server/index.js`, `lib/network/game_client.dart`, `lib/network/network_manager.dart`

### Task 2.11: 匹配 MMR

**问题:** 匹配队列纯 FIFO，无技能匹配。

**方案:**
1. 服务端维护 player MMR（基于 Elo 公式）
2. 匹配时优先匹配 MMR 差距 < 200 的玩家
3. 等待超时 > 30s 放宽 MMR 限制
4. 取消匹配通知其他队列成员

**文件:** `server/matchmaker.js`, `server/db.js`
