# Phase 01: 致命修复 — PLAN.md

## 目标

修复 5 个 P0 致命缺陷，确保游戏在任意设备上基本可玩。

## 执行顺序

**Wave 1（先做）** → 屏幕自适应（基础层，影响后续所有渲染）
**Wave 2（后做）** → HUD / 防御 / 音频 / 性能（依赖固定世界坐标）

## Wave 1: 基础层

### Task 1.1: 屏幕自适应修复 ⭐ 优先

**问题:** `onGameResize` 将屏幕像素尺寸直接设为游戏世界尺寸，导致：
- 不同设备战斗节奏完全不同（iPhone SE 横穿 2.5s vs iPad 6.8s）
- 硬编码出生点（1000）在某些设备上超出世界边界
- 弹体墙壁常量硬编码 1280×600 与实际世界尺寸不一致
- **这是之前「英雄移出屏幕」+「黑边需 Container 渐变遮挡」的根本原因**

| 设备 | 宽度 | P1出生(200) | 攻击范围80px | 横穿时间@150/s |
|------|------|------------|-------------|---------------|
| iPhone SE | 375px | 偏右 | 21%战场 | 2.5秒 |
| iPad 13" | 1024px | 贴边 | 7.8%战场 | 6.8秒 |

**方案:**
1. 保持 `stageWidth=1280, stageHeight=600` 固定游戏世界尺寸
2. 移除 `onGameResize` 中 `stageWidth = size.x; stageHeight = size.y`
3. `onGameResize` 中只更新 viewport 组件（`touchControls`, `tutorialOverlay`），不修改世界尺寸
4. 移除 `wallRight = stageWidth - 10` 等动态计算 → 恢复固定边界
5. 让 Flame 默认 viewport 自动将 1280×600 世界缩放填满任意屏幕（自动 letterbox）
6. `projectile.dart`: 移除硬编码 `wallLeft/wallRight/groundY` 静态常量 → 改为读取 `FighterGame` 的 `stageWidth/stageHeight/groundY`
7. 移除 `game_screen.dart` 中 Container 渐变黑边 workaround（不再需要）

**文件:** `lib/game/fighter_game.dart`, `lib/game/components/projectile.dart`, `lib/screens/game_screen.dart`

**依赖:** 无（本 wave 唯一任务）
**影响:** 固定世界坐标后，后续 HUD/弹体/粒子坐标计算都基于一致的 1280×600

## Wave 2: 功能层

### Task 1.2: 恢复 HUD 渲染

**问题:** HUD（血条/技能冷却/连击计数/倒计时）全部为死代码，`_renderHUD()` 被 `@unused_element` 标注且从未调用。

**方案:**
1. 在 `FighterGame.render()` 末尾恢复 `_renderHUD(canvas)` 调用
2. 删除「HUD 已在 game_screen.dart 中用 Flutter widget 渲染」的误导注释
3. HUD 坐标基于固定 1280×600 世界（Wave 1 已修复），无需额外适配

**文件:** `lib/game/fighter_game.dart`

### Task 1.3: 防御公式改造

**问题:** 防御力为固定减伤 `max(1, damage - defense)`，盾卫（38防）对貂蝉（38攻）造成「无敌状态」（需 1200 次命中才能击杀）。

**方案:**
1. 将防御公式改为百分比减伤：`finalDamage = damage * (100 / (100 + defense * 2))`
   - 盾卫 38防 → 减伤 43%，有效 HP ≈ 2100
   - 貂蝉 18防 → 减伤 26%，有效 HP ≈ 1190
2. 或使用上限机制：`effectiveDefense = min(defense, attackerAtk * 0.5)`
3. 更新 `Fighter.takeDamage()` 方法

**文件:** `lib/game/components/fighter.dart`

### Task 1.4: 移动端音频实现

**问题:** `flame_audio_impl.dart` 仅是 `debugPrint` 占位，iOS/Android 完全无声。

**方案:**
1. 使用 `flame_audio` 包的 `AudioPool` 播放音效
2. 实现 `playHit()`、`playSkill()`、`playJump()`、`playProjectile()`、`playLand()` 真实播放
3. 加载 assets/sounds/ 下的 WAV 文件
4. 修复跳跃音效每帧触发问题（改为按下时单次触发）

**文件:** `lib/game/audio/flame_audio_impl.dart`, `lib/game/components/fighter.dart`

### Task 1.5: Paint 对象缓存

**问题:** render() 循环中每帧创建 Paint 对象：粒子系统 ~20 个、弹体 MaskFilter.blur、LinearGradient Shader。

**方案:**
1. `ParticleSystem`: 用单个缓存的 Paint 对象，只更新 color 值
2. `Projectile`: 缓存 Paint + MaskFilter（或改用径向渐变替代 blur）
3. `Fighter.render()`: 缓存 shadow Paint
4. `FighterGame.render()`: 缓存 `_bgShader`，仅在 resize 时重建
5. `FighterGame._buildText()`: 预构建 Paragraph 对象并缓存

**文件:** `lib/game/components/particle_system.dart`, `lib/game/components/projectile.dart`, `lib/game/components/fighter.dart`, `lib/game/fighter_game.dart`
