# 英雄格斗 (Hero Fighter) LAN 多人联机测试方案

本文档描述如何测试英雄格斗游戏的 LAN 多人对战功能，包括设备发现、连接建立、回滚机制验证等。

## 1. 概述

### 1.1 测试目标

- 验证两台 iOS 设备在同一 WiFi 网络下的自动发现功能
- 验证 WebSocket 连接建立和房间管理
- 验证游戏过程中输入同步和回滚机制正常工作
- 验证断线重连功能

### 1.2 技术架构简述

| 组件 | 说明 |
|------|------|
| **LAN 发现** | UDP 广播 (255.255.255.255:3001) 发送 discovery 包，服务器响应 |
| **游戏连接** | WebSocket 直连 (ws://{ip}:{port}) |
| **回滚引擎** | GGPO 风格，30fps 固定帧率，环形缓冲区，延迟帧数 = 2 |
| **快照间隔** | 每 4 帧保存一次游戏快照 |
| **缓冲区大小** | 64 帧历史记录 |

## 2. 前置条件

### 2.1 硬件要求

- **两台 iOS 设备** (iPhone/iPad)
- 两台设备安装相同版本的英雄格斗游戏

### 2.2 网络要求

- 两台设备连接至**同一个 WiFi 网络**
- WiFi 网络允许设备间 UDP 广播和 WebSocket 连接
- 建议使用**本地路由器**，避免企业级网络 (可能有防火墙限制)

### 2.3 软件要求

- 游戏中已选择角色并准备就绪
- 设备已授予本地网络权限 (iOS 14+ 需要)

### 2.4 辅助工具 (可选)

- **Network Link Conditioner**: 用于模拟网络延迟和丢包，测试回滚机制
  - 路径: Xcode -> Tools -> Additional Tools -> Network Link Conditioner

## 3. 测试步骤

### 3.1 阶段一：设备发现测试

#### 步骤 1.1: 启动游戏服务器 (设备 A)

1. 在设备 A 上打开英雄格斗游戏
2. 进入多人游戏 (多人模式)
3. 选择 "创建房间" 或等待作为主机
4. 记录显示的**房间名称**和**端口号** (默认 3000)

**预期结果**: 设备 A 显示 "等待玩家加入" 或类似提示

#### 步骤 1.2: 设备 B 发现服务器

1. 在设备 B 上打开英雄格斗游戏
2. 进入多人游戏
3. 选择 "局域网发现" 或 "LAN Discovery"
4. 等待 3-5 秒扫描

**预期结果**: 
- 设备 B 应发现设备 A 创建的房间
- 房间名称显示在列表中
- 显示服务器 IP 地址和房间数

#### 步骤 1.3: 手动连接测试 (备选)

如果自动发现失败:

1. 在设备 B 上选择 "手动连接"
2. 输入设备 A 的 IP 地址 (可在设备 A 的设置中查看)
3. 输入端口号 3000

**预期结果**: 成功建立连接

### 3.2 阶段二：连接建立测试

#### 步骤 2.1: 设备 B 加入房间

1. 在设备 B 的发现列表中点击设备 A 的房间
2. 输入玩家昵称 (可选)
3. 点击 "加入"

**预期结果**:
- 设备 A 收到 "玩家已加入" 通知
- 设备 B 显示 "已加入房间"
- 双方显示相同的房间信息

#### 步骤 2.2: 角色选择

1. 设备 A 和设备 B 各选择一个英雄
2. 双方选择完毕后，点击 "准备" (Ready)

**预期结果**:
- 两边都能看到对方选择的英雄
- 显示 "双方已准备" 后，倒计时开始

#### 步骤 2.3: 游戏开始

倒计时结束后，游戏正式开始。

**预期结果**:
- 双方同时进入战斗画面
- 帧率同步为 30fps

### 3.3 阶段三：游戏同步测试

#### 步骤 3.1: 基础操作同步

1. 玩家 A 按下 "左" 方向键
2. 玩家 B 观察对手 (玩家 A) 的移动

**预期结果**: 
- 玩家 A 的角色在本地立即移动 (零延迟)
- 玩家 B 屏幕上的对手几乎同步移动 (延迟 1-3 帧内)

#### 步骤 3.2: 攻击同步

1. 玩家 A 发起普通攻击
2. 观察攻击动作和伤害数字

**预期结果**:
- 攻击动作在双方屏幕同步显示
- 伤害数字一致

#### 步骤 3.3: 技能同步

1. 玩家 A 释放技能
2. 观察技能特效和命中效果

**预期结果**:
- 技能特效同步显示
- 命中效果 (击退、伤害) 双方一致

### 3.4 阶段四：回滚机制测试

#### 步骤 3.4.1: 启用 Network Link Conditioner (可选)

在 Mac 上:
1. 打开 Xcode -> Tools -> Network Link Conditioner
2. 添加配置: 延迟 100ms, 丢包率 5%
3. 在测试设备上启用该配置文件

#### 步骤 3.4.2: 观察回滚现象

1. 在高延迟情况下进行对战
2. 观察以下现象:

**预期结果 (回滚发生)**:
- 角色位置**短暂回退** (可能出现闪烁)
- 之后快速**重新模拟**到当前位置
- 玩家可能感到轻微"卡顿"后继续

**如何确认是回滚而非卡顿**:
- 回滚: 角色位置明显回到之前的位置，然后快速前进
- 网络卡顿: 角色静止不动

#### 步骤 3.4.3: 统计数据验证

在代码中添加调试输出可以查看:
- `rollbackCount`: 回滚次数
- `resimulatedFrames`: 重新模拟的帧数

**预期结果**:
- 网络条件越差，回滚次数越多
- 延迟帧数设置为 2 时，回滚应该相对较少

### 3.5 阶段五：断线测试

#### 步骤 5.1: 模拟断线

**方法一**: 手动断开
1. 在对战过程中，关闭设备 A 的 WiFi
2. 观察设备 B 的反应

**方法二**: 离开范围
1. 将设备 A 拿到 WiFi 范围外

**预期结果**:
- 设备 B 显示 "对手已断开连接" 或类似提示
- 游戏暂停或显示等待画面

#### 步骤 5.2: 重新连接 (测试重连机制)

1. 恢复设备 A 的网络连接
2. 观察是否自动重连

**预期结果** (如果实现了重连):
- 短时间断线 (< 30秒) 可能自动重连并恢复游戏
- 长时间断线则房间解散

### 3.6 阶段六：房间管理测试

#### 步骤 6.1: 离开房间

1. 在对战结束后或进行中，点击 "离开房间"
2. 观察双方状态

**预期结果**:
- 离开的玩家返回主菜单
- 房间内其他玩家收到通知

#### 步骤 6.2: 房间解散

1. 房主离开房间
2. 观察房间状态

**预期结果**: 房间自动解散，其他玩家收到通知

## 4. 故障排除

### 4.1 设备发现失败

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 找不到服务器 | 设备不在同一网络 | 确保连接同一 WiFi |
| 找不到服务器 | UDP 端口被阻止 | 检查路由器设置，尝试手动连接 |
| 找不到服务器 | iOS 本地网络权限未授权 | 在设置中允许本地网络访问 |

### 4.2 连接失败

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| WebSocket 连接失败 | IP 地址错误 | 确认设备 IP 地址正确 |
| 连接超时 | 防火墙阻止 | 暂时关闭防火墙测试 |
| 连接被拒绝 | 服务器未启动 | 确认设备 A 已创建房间 |

### 4.3 同步问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 角色位置明显不同步 | 网络延迟过高 | 改善网络环境 |
| 频繁回滚 | 网络不稳定 | 使用 Network Link Conditioner 测试，延迟帧数是否合适 |
| 游戏不同步 | 帧率不一致 | 确保游戏锁定 30fps |

### 4.4 回滚异常

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 回滚后画面闪烁 | 快照恢复与渲染冲突 | 检查 `GameSnapshot.restoreTo()` 实现 |
| 回滚后位置错误 | 确定性模拟失败 | 检查随机数生成是否依赖帧号 |
| 回滚次数为 0 但感觉卡顿 | 延迟帧数不足 | 增加 `delayFrames` (默认 2) |

### 4.5 性能问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 帧率下降 | 回滚重算过多 | 减少 `snapshotInterval` 或增加 `bufferSize` |
| 内存占用高 | 缓冲区过大 | 减小 `bufferSize` (默认 64) |

## 6. 调试技巧

### 6.1 启用调试日志

在代码中添加以下输出:

```dart
// 在 RollbackEngine 中
print('Frame: $_frame, Rollback: $_rollbackCount, Resim: $_resimulatedFrames');
print('Remote delay: $remoteDelay');
```

### 6.2 监控网络状态

使用 Flutter 开发者工具或添加:

```dart
// 监控连接状态
networkManager.onConnectionStateChanged = (state) {
  print('Connection state: $state');
};
```

### 6.3 帧数据验证

在测试中记录双方每帧的输入和位置，验证:
- 输入序列一致性
- 快照恢复后状态一致性
